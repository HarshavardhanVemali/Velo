# Security Policy

## Supported Versions

Currently, only the latest release of Velo is supported with security updates. We recommend always running the latest version from the `main` branch.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

We take the security of Velo seriously. If you discover a security vulnerability, please do NOT report it by opening a public GitHub issue. 

Instead, please report it via one of the following methods:

1. **Email:** Send a detailed report to the core maintainers.
2. **GitHub Security Advisory:** Submit a private vulnerability report through GitHub's "Security" tab if available.

### What to include in your report

Please provide as much information as possible to help us quickly identify and resolve the issue:

* **Description:** A clear summary of the vulnerability.
* **Steps to Reproduce:** Exact steps to trigger the vulnerability. Include any specific configuration or environment variables if relevant.
* **Impact:** An explanation of how an attacker could exploit this and what the potential impact would be.
* **Proposed Solution (Optional):** If you have a fix or mitigation strategy, please share it.

### What happens next?

Once you submit a report, a maintainer will:
1. Acknowledge receipt of your report within 48 hours.
2. Investigate the issue to confirm the vulnerability.
3. Work on a patch and coordinate a release date with you.
4. If applicable, coordinate a public CVE or security advisory after the patch is released.

Thank you for helping keep Velo and our community secure!
